# v0.2.0 — Write Operations, SSE Transport & Streaming

**Status:** Planning
**Target:** Next minor release across all packages

---

## Theme

Evolve from read-only to selective write operations, add SSE transport for remote hosting, and introduce streaming responses for large payloads.

---

## Features

### 1. Write operations (always available, consent via MCP annotations) — [Full Spec](v0.2.0-write-operations.md)

Add ~45 write/mutation tools across all 11 packages. No feature flag — writes are always available. User consent is handled by MCP tool annotations (`readOnlyHint: false`) which prompt the MCP client to ask the user before executing.

**Implementation phases:**
1. **Sentry, Linear, PagerDuty** — resolve issues, create issues, acknowledge incidents
2. **Stripe, HubSpot, Intercom** — refunds, CRM operations, conversation replies
3. **Notion, Google Workspace, Shopify, Datadog, Vercel** — content, calendar, orders, monitors

**Design principles:**
- No flags — write tools always registered alongside read tools
- User consent via MCP `annotations` (`readOnlyHint`, `destructiveHint`, `idempotentHint`)
- Write tools live in separate files (`<entity>.write-tools.ts`)
- No hard deletes in v0.2.0 (archive/soft-delete only)
- Every write tool returns the updated resource for confirmation
- Clear, actionable error messages for write failures

### 2. SSE (Server-Sent Events) transport

Add HTTP/SSE transport alongside existing stdio, enabling remote hosting and multi-client access.

- Implement SSE transport using `@modelcontextprotocol/sdk` built-in support
- Add `--transport=sse --port=3001` CLI flags
- Support `MCP_TRANSPORT=sse` and `MCP_PORT=3001` env vars
- Add CORS configuration for browser-based clients
- Shared implementation in a new `packages/transport-core` or via oauth-core extension

### 3. Streaming responses for large payloads

For tools that return large datasets (metric queries, search results, log streams):

- Implement progressive result streaming where supported by the MCP protocol
- Add pagination helpers in each package's service layer
- Add `limit` parameter to list/search tools that don't already have one

### 4. OAuth-core improvements

- Add token encryption at rest (currently plaintext JSON)
- Add multi-account support (store tokens per account/workspace)
- Add `auth status` CLI command to check token validity

## Decisions

- [x] Write operations: **within existing package** (not separate packages). One package per service, no dependency sprawl.
- [x] No feature flag: writes always available. User consent handled by **MCP tool annotations** (`readOnlyHint: false` triggers client-side confirmation prompts).
- [x] Existing read tools: annotated with `readOnlyHint: true` (default fallback, no changes to existing schemas required).

## Open questions
